APACHE + PHP 8.4 + MARIADB + PHPMYADMIN SETUP NOTES
Lubuntu 22.04 — April 2026
======================================================

OVERVIEW
--------
Ubuntu and Lubuntu do not ship modern PHP versions in their default repositories.
Attempting to install php8.3 or php8.4 directly will fail with "unable to locate package".
The fix is to add the Ondrej Sury PPA, which is the standard third-party repository
for up-to-date PHP builds on Ubuntu and Debian systems.


STEP 1 — UPDATE AND INSTALL APACHE
-----------------------------------
sudo apt update
sudo apt install -y apache2
sudo systemctl enable apache2
sudo systemctl start apache2

Test by opening http://127.0.0.1 in Firefox. You should see the Apache default page.


STEP 2 — ADD THE ONDREJ PHP PPA (required for PHP 8.x)
--------------------------------------------------------
Ubuntu's default repos only carry older PHP versions. To get PHP 8.4 you must
add the Ondrej Sury PPA as an additional package source before installing.

sudo apt install -y software-properties-common
sudo add-apt-repository ppa:ondrej/php
sudo apt update

Once the PPA is added, modern PHP packages become available via apt normally.


STEP 3 — INSTALL PHP 8.4
--------------------------
sudo apt install -y php8.4 php8.4-cli php8.4-common php8.4-mysql php8.4-xml php8.4-mbstring php8.4-curl libapache2-mod-php8.4

Enable the module and restart Apache:
sudo a2enmod php8.4
sudo systemctl restart apache2

Verify the install:
php -v

You should see PHP 8.4.x reported in the output.


STEP 4 — INSTALL MARIADB
--------------------------
sudo apt install -y mariadb-server
sudo systemctl enable mariadb
sudo systemctl start mariadb
sudo systemctl status mariadb

Look for "Active: active (running)" in the status output.


STEP 5 — SECURE MARIADB
-------------------------
sudo mysql_secure_installation

When prompted answer as follows:
- Enter current password for root: press Enter (blank, no password set yet)
- Switch to unix_socket authentication: N
- Change the root password: Y
- New password: teacher123
- Remove anonymous users: Y
- Disallow root login remotely: Y
- Remove test database: Y
- Reload privilege tables: Y

Then force root to use password authentication (required for phpMyAdmin):
sudo mysql -u root

Inside the MariaDB shell run:
ALTER USER 'root'@'localhost' IDENTIFIED VIA mysql_native_password USING PASSWORD('teacher123');
FLUSH PRIVILEGES;
EXIT;


STEP 6 — INSTALL PHPMYADMIN
-----------------------------
sudo apt install -y phpmyadmin

During the installer a blue screen appears. Answer as follows:
- Web server to configure: select apache2 (spacebar to select, then Enter)
- Configure database with dbconfig-common: Yes
- MySQL application password: press Enter (auto-generate)

Link phpMyAdmin into Apache and reload:
sudo ln -s /etc/phpmyadmin/apache.conf /etc/apache2/conf-available/phpmyadmin.conf
sudo a2enconf phpmyadmin
sudo systemctl reload apache2

Access phpMyAdmin at http://127.0.0.1/phpmyadmin
Login: root / teacher123


STEP 7 — ENABLE MOD_REWRITE AND CONFIGURE APACHE
--------------------------------------------------
sudo a2enmod rewrite
sudo systemctl restart apache2

Edit the default site config to allow .htaccess overrides:
sudo nano /etc/apache2/sites-available/000-default.conf

Inside the VirtualHost block add:
<Directory /var/www/html>
    Options Indexes FollowSymLinks
    AllowOverride All
    Require all granted
</Directory>

Restart Apache after any config change:
sudo systemctl restart apache2


STEP 8 — CREATE THE WEB GROUP
-------------------------------
Create a shared group so kim and teacher can both write to /var/www without sudo.

sudo groupadd web
sudo usermod -aG web kim
sudo usermod -aG web teacher

Set ownership and permissions on /var/www:
sudo chown -R www-data:web /var/www
sudo chmod -R 775 /var/www
sudo chmod g+s /var/www

The g+s (setgid) flag ensures all new files and folders created inside /var/www
automatically inherit the web group. Without it, files created by kim would be
owned by kim:kim and teacher could not edit them.

Log out and back in for group membership to take effect, then verify:
groups kim
groups teacher


STEP 9 — /etc/hosts LOCAL ROUTING
-----------------------------------
sudo nano /etc/hosts

Add these lines:
127.0.0.1  home
127.0.0.1  homework

http://home will now resolve to /var/www/html/index.html
http://homework will resolve to /var/www/html/ (point to homework.html via vhost or direct link)

Copy your project files into place:
sudo cp home.html /var/www/html/index.html
sudo cp homework.html /var/www/html/homework.html


QUICK SERVICE COMMANDS
-----------------------
sudo systemctl start apache2
sudo systemctl stop apache2
sudo systemctl restart apache2
sudo systemctl status apache2

sudo systemctl start mariadb
sudo systemctl stop mariadb
sudo systemctl restart mariadb
sudo systemctl status mariadb


NOTES
------
- PHP 8.5 is the latest version as of 2026 but 8.4 is the more stable choice for Ubuntu 22.04.
- The Ondrej PPA only supports Ubuntu LTS releases (22.04, 24.04).
- MariaDB 10.6 is the version shipped with Ubuntu 22.04 and is fully MySQL-compatible.
- www-data must remain the file owner so Apache can serve files correctly.
  The web group sits alongside it for developer write access.
- phpMyAdmin lives at /usr/share/phpmyadmin and is symlinked into Apache config.
