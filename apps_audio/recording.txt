Yes, there are several ways to do this in Lubuntu. Since Lubuntu uses a lightweight setup (traditionally based on **PulseAudio** or **PipeWire**), you have excellent command-line and GUI options.

In Linux, the concept you are looking for is a **"Monitor" source**. This is a virtual input that "listens" to what is being sent to your speakers (Line Out).

---

## 1. Quick Terminal (Bash) Solutions
These are the fastest way to start recording without installing heavy software.

### Using `ffmpeg` (Recommended)
`ffmpeg` is incredibly robust and usually pre-installed. You can record your system output directly to a `.wav` file with one command:
```bash
ffmpeg -f pulse -i default.monitor output.wav
```
* **Note:** If `default.monitor` doesn't work, run `pactl list short sources` to find the exact name of your monitor device (it usually ends in `.monitor`).

### Using `parec` (PulseAudio Native)
If you want a very "raw" PulseAudio approach:
```bash
parec --monitor-stream=$(pactl list sink-inputs | awk '$1 == "index:" {print $2}') --file-format=wav output.wav
```

---

## 2. GUI Application: Audio Recorder
If you prefer a simple window with a "Record" button, **Audio Recorder** is the best lightweight tool for Lubuntu. It can record specifically from your "System Output."

**To install it:**
```bash
sudo add-apt-repository ppa:audio-recorder/ppa
sudo apt update
sudo apt install audio-recorder
```
* **Why use it?** It has a built-in timer, auto-start/stop based on voice/sound detection, and an easy dropdown to select "Speaker" or "Line Out" as the source.

---

## 3. Python Scripting
If you want to build this into a custom tool, you can use the `SoundCard` library or `PyAudio`. Here is a simple example using `SoundCard`, which is often easier for capturing "loopback" (output) audio than standard PyAudio.

**First, install the library:**
```bash
pip install SoundCard numpy SoundFile
```

**The Script:**
```python
import soundcard as sc
import soundfile as sf

# Set recording parameters
SAMPLE_RATE = 44100
DURATION = 5  # seconds
OUTPUT_FILE = "output.wav"

# Get the loopback (monitor) speaker
with sc.get_microphone(id=str(sc.default_speaker().name), include_loopback=True).recorder(samplerate=SAMPLE_RATE) as mic:
    print(f"Recording for {DURATION} seconds...")
    data = mic.record(numframes=SAMPLE_RATE * DURATION)
    
# Save to WAV
sf.write(OUTPUT_FILE, data, SAMPLE_RATE)
print(f"Saved to {OUTPUT_FILE}")
```

### Comparison at a Glance

| Method | Ease of Use | Best For... |
| :--- | :--- | :--- |
| **FFmpeg** | Medium | One-off captures via terminal. |
| **Audio Recorder** | Easy | Users who want a GUI and "Auto-stop" features. |
| **Python** | Hard | Automating tasks or building custom apps. |
| **Audacity** | Medium | Recording AND editing the file immediately. |

### A Quick Tip for Lubuntu Users
If you find that the audio isn't being picked up, install `pavucontrol` (**PulseAudio Volume Control**). Open it while you are recording, go to the **Recording** tab, and ensure your recording application is set to capture from **"Monitor of [Your Device Name]"**.
